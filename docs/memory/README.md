# Simple Memory System

Last synchronized from `GOVERNANCE.md`: 2026-02-27

## Goal
Provide a lightweight, durable memory system that lets agents execute delivery work without repeatedly re-parsing `GOVERNANCE.md`.

## File Map
- `core-principles.md`: distilled governance rules used during coding.
- `workflow.md`: normal workflow from idea to deployment.
- `checklists.md`: concise phase and release checklists.
- `records/decision-log.md`: key decisions and rationale.
- `records/testing-log.md`: tests executed and outcomes.
- `records/documentation-log.md`: docs created/updated.
- `records/release-log.md`: release gate evidence.

## Usage Rules
1. Start each task by reading principles, workflow, and checklists.
2. Keep records current as work progresses.
3. If governance and memory conflict, update memory first, then continue.
4. Keep entries short, dated, and evidence-based.

## Alignment with `agents.md`
This repository follows the `AGENTS.md`-based instruction model described at [agents.md](https://agents.md/).

`AGENTS.md` defines the operational contract and startup routine, while `docs/memory/` provides the working memory and evidence records used during task delivery.
